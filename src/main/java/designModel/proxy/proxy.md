### 代理模式
代理模式：通过持有被代理类的对象来完成被代理者的功能，实现被代理者与用户的解耦。这样做到好处是可以保持被代理类不变动的情况下做到增加额外的功能，符合开闭原则。注意与适配器模式的区别，适配器是处理两个接口不兼容的问题。    
代理模式分为两种：静态代理和动态代理。  
- 静态代理：在使用静态代理时,被代理对象与代理对象需要一起实现相同的接口或者是继承相同父类，因此要定义一个接口或抽象类.
